# 我的通用诉求

## 核心场景

我会在多台设备之间切换使用（手机、iPad、不同电脑），所以做任何产品都要考虑**跨端访问**和**数据同步**。不要给我做只能在一台机器上跑的东西。

## 产品要求

- 轻量化：不要搞重型框架和复杂架构，能简单就简单
- 跨端可用：手机、平板、电脑都能用，优先 PWA 或 Web 应用
- 数据同步：多设备之间数据要能同步，可以用 Supabase、GitHub 仓库、云服务器等方式
- 自动化：周期性任务（如每日更新内容、定时抓取信息）必须自动运行，不能依赖我手动触发

## 我有的资源

- **GitHub 账号**：代码托管、GitHub Pages 静态部署、GitHub Actions 做定时任务
- **MiniMax Token Plan**（199元/月）：AI API 调用，用于自动化内容生成
- **轻量云服务器**（2C4G）：需要后端的场景用这个
- **域名 eric-ai.top**：通过 Cloudflare 管理 DNS 和 HTTPS
- **个人电脑**：日常开发和维护

## 自动化原则

- 周期任务用 GitHub Actions 跑在云端，不依赖我的电脑是否开机
- API Key 等敏感信息存 GitHub Secrets 或服务器环境变量，不硬编码
- 自动化失败了要能自愈（次日重试），不需要我介入

## 成本控制

- 能免费就免费（GitHub Pages、Cloudflare、GitHub Actions 免费额度）
- 已有的付费资源（MiniMax、服务器、域名）可以充分利用
- 不要给我推荐新的月费服务，除非真的没有替代方案
